# 安全政策

## 🔒 支持版本

| 版本 | 支持状态 |
|------|----------|
| 0.1.x | ✅ 支持 |

## 📣 报告安全漏洞

如果你发现了安全漏洞，请**不要**通过公开 Issue 报告。

请通过以下方式私下报告：

1. 发送邮件至项目维护者
2. 使用 GitHub 的 [Security Advisories](https://github.com/jcy321/opennovel-hub/security/advisories) 功能

我们承诺：

- 在 48 小时内确认收到报告
- 在 7 天内提供初步评估
- 及时修复并发布安全更新
- 在修复后公开致谢（如果你愿意）

## 🛡️ 安全最佳实践

### API 密钥管理

OpenNovel 使用环境变量管理敏感信息：

```bash
# 设置 LLM API 密钥
export GLM5_API_KEY="your-api-key"
```

**切勿**将 API 密钥硬编码在代码中或提交到版本控制。

### 部署安全

- 使用 HTTPS 加密通信
- 定期更新依赖包
- 限制服务端口的外部访问
- 使用防火墙规则保护服务

## ⚠️ 已知限制

- 当前版本为早期开发版，不建议用于生产环境
- 用户数据存储在本地，请自行做好备份
- LLM API 调用需要外部网络连接
